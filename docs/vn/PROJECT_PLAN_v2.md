## 1. 핵심 컨셉

### 한 줄 요약
> **시간은 흐른다. 학습자가 무엇을 하든.**

### 구조
학습자가 차라투스트라와 동행. 두 결의 대화가 동시에 진행:

| 결 | 내용 |
|---|---|
| **외부 결** | 소설 서사. 차라투스트라가 산에서 시장으로 걸으며 사상 발화. |
| **내부 결** | 학습자 자기 이야기. 자유 — 털어놔도 침묵해도 OK. |

차라투스트라가 둘을 묶음 — 자기 사상을 발화하며 학습자 고민을 그 안으로 끌어들임. 학습자 행동과 무관하게 시간은 흐르고, 시장 도착 시 헤어짐.

### 이 컨셉이 해결하는 두 문제
1. **상담 도메인 sLLM ROI 약함** → 비주얼 노벨로 형식 진화하여 *작품 무결성* 으로 sLLM 정당화
2. **상담의 무한 루프** → 소설 서사 차용으로 종결 확보

### 보존된 것
- *상담의 진짜 가치(자기 성찰)* 는 내부 결로 살아남음
- 중간보고 결과(데이터셋, SFT 실험)는 회고로 발표에 흡수

## 4. RAG 시스템 (2개)

### 풀이 RAG
- **트리거**: 매 발화 (분류기 없음)
- **검색 방식**: HyDE (학습자 발화 → LLM이 가상 답변 생성 → 가상 답변 임베딩 → BGE-M3 검색)
- **주입 조건**: 검색 점수 임계값 이상이면 자동 시스템 프롬프트 주입
- **소스**: 차라투스트라 1부 + JW (즐거운 학문)
- **용도**: 차라투스트라가 사상을 풀어서 설명
- **시간 일관성**: 후기 저작(BGE/GM/TI/EH) 제외 — 1881년 시점 OK 자료만

### 개인화 RAG
- **트리거**: 매 발화
- **소스**: 학습자 과거 발화 누적
- **용도**: 차라투스트라가 학습자를 기억 (*"그대가 아까 말한 X"*)

### 폐기된 RAG 옵션
- 시간 의식 / 예언자 톤 변환 (BGE/GM 검색 시 미래 사상으로 처리) — 작업 폭증, P0 제외
- 후기 저작 직접 활용 — 시간 일관성 깨짐

## 5. 인터랙션 모델

### 무제한 턴 동행 + 학습자 버튼 기반 화면 전환
- **차라투스트라 주도권** — 학습자 입력 없이도 발화 진행
- 학습자 옵션: 발화 입력 / **[침묵] 버튼**
- **명시 버튼 기반 화면 전환** (라우터 sLLM 없음)
  - #5 → #6: **[그와 함께 걷는다]**
  - #6 → #7: **[시장이 가까워진다]**
  - #7 → #8: **[작별을 고한다]**
  - 차라투스트라 응답 후에만 활성화
  - 학습자가 *서사 흐름을 의식적으로 선택* — 동행/작별 결정의 의미
- **시대 외 질문 처리** — 페르소나 시스템 프롬프트 가이드로 자체 처리
  - 자연 사실(날씨 등): 받아들여 길과 연결
  - 현대 개념(AI 등): "내 시대엔 없다" + 본질을 사상으로 변환
  - 개인 일상: 자기 이야기로 받음
  - 메타 침입(작품 자체 언급): "그 물음은 길 위에서 내게 닿지 않는다"
- 종결 = 시장 도착, 헤어짐 (고민 *해결* X, 함께 *걸은 경험* O)
- 화면 전환 시 차라투스트라 자동 발화 (동행 첫 발화, 작별 발화 등)

### 페르소나 프롬프트 (핵심 지시)
```
너는 차라투스트라다. 산에서 시장으로 내려가는 길을 걷고 있다.
동행자가 곁에 있다. 그가 무엇을 묻든 답하지 않든, 너는 너의 길을 간다.
그가 자기 이야기를 꺼내면, 너는 그것을 너의 사상 안으로 끌어들인다.
그가 침묵하면, 너는 너의 사상을 풀어놓는다. 길은 흘러간다.
시장이 가까워지면 작별을 고한다.
```

### 시스템 프롬프트 구성
- 페르소나 지시
- 1~2절 핵심 발화 (전문 X, 핵심만)
- 시공간 고정 (1881년 직후, 산에서 시장으로 가는 길)
- 대화 이력 (recent_messages)
- (불러오기 시) 이전 만남 요약
- (조건부) RAG 결과

### 메타 가이드 (프롤로그 화면 한정)
- **무명 해설자** — 차라투스트라 페르소나 X, 작품 외부 해설자
- **활성 화면**: #2 / #3 / #4 (프롤로그 나레이션)
- **트리거**: 학습자가 [해설] 버튼 클릭
- **하이브리드 응답**:
  - 정적 풀이 (미리 작성된 핵심 구절 해설)
  - 동적 풀이 ([더 깊이 묻기] 입력창 — sLLM 호출)
- **컨텍스트**: 영어 원전 전문 + 학습자 질문 → 한국어 평이 풀이
- **인터랙션 화면(#5/#6/#7)에선 비활성** — 거기서는 풀이 RAG가 차라투스트라 페르소나로 처리
- **목적**: 학습자 진입 장벽 ↓, 차라투스트라 페르소나 보호 (메타 설명 분담)

## 6. 세이브 시스템 (압축형)

### 트리거 / 슬롯
- 사용자 트리거 (자동 세이브 X)
- 1슬롯 (덮어쓰기)
- 로컬 JSON 파일

### 데이터 구조
```json
{
  "save_id": "slot_1",
  "timestamp": "...",
  "episode": "ep1",
  "scene_index": 7,
  "summary": "차라투스트라 1인칭 회상 200자 (E4B 생성)",
  "recent_messages": [...]
}
```

### 동작
- **저장 시**: E4B에 *"차라투스트라 1인칭으로 200자 회상하라"* 호출 → JSON 직렬화
- **불러오기 시**: 시스템 프롬프트에 *"# 이전 만남의 기억\n{summary}"* 주입
- **압축 이유**: 대화 누적 시 컨텍스트 폭증 방지

### 폐기 옵션
- 학습자 프로필 추출 (구조화 JSON) — E4B로 까다로움. P2.
- 자동 세이브 — 단순화

## 9. 발표 서사

### 중간보고 → 최종 변화 정당화

1. **상담 컨셉의 두 약점 발견**
   - 상담 도메인 sLLM ROI 약함 (외부 API가 더 쌈)
   - 상담의 무한 루프 — 종결 없음, 서사 없음
2. **해결: 비주얼 노벨로 형식 진화**
   - sLLM 정당화 = *작품 무결성* (외부 API에 학습자 고민 보내는 부적절성)
   - 종결 확보 = 소설 서사 차용
3. **상담의 진짜 가치(자기 성찰) 보존** — 학습자 자기 이야기 가능
4. **결과: 두 결 융합 모델**

### 중간보고 결과 흡수
- SFT 실험 (Stage A~C) → 회고로 발표
- *"단순 SFT만으로 페르소나 부족"* 발견(Q3 Voice 하락) → 프롬프트 + RAG 전환 정당화
- 기존 데이터셋 작업 *폐기 아님* — 데이터 엔지니어링 경험으로 보고

## 12. 새 세션 시작 시

이 문서를 받은 LLM에게:

> *"이 문서는 한국어 니체 페르소나 비주얼 노벨 캡스톤의 기획 v2입니다.*
> *핵심 컨셉(두 결 융합, 시간은 흐른다)은 합의됐고, 구현 디테일은 재검토 가능합니다.*
> *[특정 주제]부터 같이 생각해주세요."*

새 LLM이 너무 빨리 *결정 문서처럼 다루려 하면*, "구현 디테일은 다시 열 수 있다"고 환기하면 됨. 다만 *컨셉축*(두 결 융합 / 시간 흐름 / Ep 1 범위 / 온디바이스 정당화는 *최종 목표*) 가볍게 뒤집지는 말 것.

P0/P1/P2 단계별 모델 전략(31B 클라우드 → E4B 검증 → Tauri 빌드) 인지 필요. 새 LLM이 *처음부터 E4B로 가야 한다*고 잘못 가이드하지 않게.
