## 2. Ⅰ. 정적 나레이션 (#1~#4, #8, Ep 2 #1~#3)

### 2.1 단락 표시 방식
**교체**. 이전 단락 페이드아웃 → 새 단락 페이드인. 누적 X.
- 다중 줄 인용(예: *"위대한 별이여..."*)은 *한 단락 안에* 한 번에 표시.
- 단락 안 빈 줄(`\n\n`)은 단락 구분이 아니라 *호흡* — 같은 단락 안 시각적 여백.

### 2.2 클릭 인터랙션
- 화면 어디 클릭해도 / Space / Enter → 다음 단락
- 우상단 메뉴 영역 + 좌하단 [해설] 버튼 영역은 클릭 무효 (이벤트 stopPropagation)
- 텍스트 박스 우하단 ▼ 인디케이터 (블링크 1.4s, 다음 단락 대기 신호)
- 마지막 단락 후 클릭 → 다음 화면 페이드아웃

### 2.3 [해설] 버튼
- 좌하단 위치 (#2~#4, #8, Ep 2 #1~#3 한정)
- 첫 단락 페이드인 직후부터 항상 활성
- 패널 열려있는 동안 ▼ 인디케이터 비활성, 클릭 무효 (다음 단락으로 못 넘어감)
- 패널 닫으면 보던 단락 그대로 복귀

### 2.4 페이드 타이밍
- 화면 진입/이탈: 600ms (단, **#4 → #5는 800ms** — 분위기 전환점)
- 단락 전환: 280~300ms (out-in 동시)
- #8 엔딩 일러스트 정적 정지: 5초 (텍스트 페이드 전)

### 2.5 차라투스트라 발화 인용 (나레이션 안)
- 단락 안에서 *살짝 들여쓰기 + italic*으로 시각 구분
- 같은 단락 호흡으로 처리 (단락 분리 X)

### 2.6 #8 엔딩 카드 (Ep 1)
**EP1_TEXT_AND_PROMPTS.md의 [세이브하고 종료]/[세이브 없이 종료] 폐기**. 다음으로 변경:

## 3. Ⅱ. 인터랙션 (#5~#7, Ep 2 #4)

### 3.1 발화 표시
**누적 스크롤**. 텍스트 박스(하단 영역) 내부에서 자동 스크롤. 메시지 사이 12px 간격.

### 3.2 화자별 시각 분리
| 화자 | 위치 | 색상 | 화자명 | 본문 |
|---|---|---|---|---|
| 차라투스트라 | 좌측 | 잉크 #2a1f14 | "차라투스트라" Georgia 11px letter-spacing 0.18em | Georgia 14~15px line-height 1.7 |
| 학습자 | 우측 | 옅은 잉크 #6b5f4f | "그대" 같은 양식, 우측 정렬 | 우측 정렬 |

### 3.3 [침묵] 시각화
- 학습자 영역에 *"…"* (말줄임표 3개, italic, 옅은 잉크) 메시지로 누적
- 즉시 차라투스트라 응답 sLLM 호출 (시스템 프롬프트에 침묵 지시 주입)
- 학습자도 *침묵을 선택했다*는 행위 기록을 시각으로 확인

### 3.4 입력창 / 버튼 배치
화면 하단 영역 (텍스트 박스 *바깥*에 별도 행):

```
[ 차라투스트라 발화 누적 박스 — 자동 스크롤 ]
─────────────────────────────────────
[ 입력창 (textarea, 1줄→2줄 자동 높이) ]
[발화하기]  [침묵]              [화면 전환 버튼 →]
```

- 좌측: 발화 액션
- 우측: 화면 전환 버튼 (현재 화면별 라벨 다름 — §3.6)

### 3.5 입력 / 버튼 정책
- **발화 길이 제한**: 500자. 우하단 카운터 `217 / 500`. 450자 초과 시 색 진해짐. 500자 초과 시 [발화하기] 비활성.
- **빈 입력 [발화하기]**: 비활성 (회색)
- **[침묵]**: 항상 활성 (입력창 비어있는 게 침묵의 신호가 아니라 *행위*가 침묵)
- **응답 스트리밍 중**: 입력창 disabled, [발화하기]/[침묵]/화면 전환 버튼 모두 비활성. placeholder는 *"…"* 신호로 변경.
- **응답 지연 > 3초**: 차라투스트라 영역 좌측에 italic 작은 *"…"* 인디케이터 (RAG/sLLM 호출 시간 흡수)
- Enter → 전송 / Shift+Enter → 줄바꿈

### 3.6 화면 전환 버튼

| 화면 | 라벨 |
|---|---|
| Ep 1 #5 만남 | **[그와 함께 걷는다 →]** |
| Ep 1 #6 동행 | **[시장이 가까워진다 →]** |
| Ep 1 #7 시장 원경 | **[작별을 고한다 →]** |
| Ep 2 #4 학습자 재회 | **[작별을 고한다 →]** |

### 3.7 화면 전환 버튼 활성 조건
- **비활성 (옅은 잉크 #b8ad9b, hover 없음, cursor not-allowed)**:
  - 차라투스트라 응답 스트리밍 중
  - 학습자 발화 0회 (1회도 안 한 상태로는 화면 못 넘김)
- **활성 (잉크 #2a1f14)**:
  - 차라투스트라 응답 완료 + 학습자 발화 ≥ 1회

[침묵]도 *발화 1회*로 카운트 (행위 기록).
학습자 입력 중에도 화면 전환 버튼은 활성 유지 (입력 안 보내고 바로 전환 가능).

### 3.8 자동 발화 페이드 시퀀스 (화면 진입 시)
화면 페이드인 600ms 완료 후:
1. **200ms 정적** (책장 넘기는 호흡)
2. 차라투스트라 화자명 페이드인 (300~320ms)
3. 본문 토큰 스트리밍 시작 (vLLM 응답 그대로, 32ms/char 시뮬)
4. 스트리밍 완료 → 입력 활성

### 3.9 화면별 기조 차이
시스템 프롬프트의 *현재 상황* 섹션이 화면별로 다름. UI 컴포넌트는 동일.
- #5 만남: 첫 발화 *"그대. 어디서 왔는가."* 고정. 탐색 결.
- #6 동행: 자동 발화 = 사상 흘리기 / 풍경 / 짧은 질문. 융합 결.
- #7 시장 원경: 자동 발화 = 작별 결 비춤 (직접 작별 X). [작별을 고한다] 클릭 후 명확한 작별.
- Ep 2 #4: 자동 발화 = Ep 1 요약 + 개인화 RAG 호출 결과 반영. 재회 결.

### 3.10 #7 / Ep 2 #4 작별 종결
[작별을 고한다] 클릭 시:
1. 마지막 작별 발화 sLLM 호출 (`EP1_TEXT_AND_PROMPTS.md` §5 #7 참조)
2. 학습자 마지막 응답 1회 가능 (선택, 입력 안 해도 OK)
3. 페이드아웃 → 엔딩 카드

### 3.11 차라투스트라 발화 길이 제어
- UI 차원 잘라내기 X. sLLM 출력 그대로 표시.
- 안전장치는 vLLM `max_tokens=1500`에서 강제 종료 (P0 vLLM 설정).

### 3.12 메타 침입 / 트롤 케이스
- UI 차원 처리 X. 시스템 프롬프트(§2 시대 외 질문 가이드)가 처리.
- 학습자가 반복적으로 메타 침입해도 작품은 그대로 흐름. 강제 종료 / 경고 없음.

## 4. Ⅲ. 해설 모드 (#2~#4 / Ep 2 #1~#3)

### 4.1 패널 동작
**우측 슬라이드 50% 패널**. 모달/전체화면 X — *책 삽화 페이지의 측주(marginalia)* 결.
- 슬라이드 인: 380ms ease
- 배경: #ebe0c8 (frame보다 살짝 어두운 세피아)
- 좌측 0.5px 잉크 보더로 frame과 분리

### 4.2 정적 풀이 (손글씨)
**황철희 직접 작성. sLLM X.**

분량: 한 화면당 200~350자.
구조 (3섹션 고정):
```
[제목]
서문 1절 — 산 정상   (Georgia, letter-spacing)

[구분선]
─────────────────────

[장면 한 줄 요약]
이 화면은 차라투스트라가 10년의 고독을 마치고 하산을 결심하는 장면입니다.

[▸ 핵심 구절 풀이]
(인용 1~2개 — italic + 좌측 들여쓰기 + 한 줄 풀이씩)

[▸ 더 깊이 묻기]
[ 입력창 ]
[ 묻기 ]
```

### 4.3 동적 풀이 ([더 깊이 묻기])
- **트리거**: 학습자가 입력창에 질문 작성 후 [묻기] 클릭
- **응답 영역**: 입력창 *아래*에 누적. 정적 풀이는 위에 유지.
- **응답 길이**: 300~500자 (RAG 컨텍스트 들어오므로 풀어 설명)
- **톤**: 무명 해설자. *"~입니다"* 평문체. 차라투스트라 1인칭 X.
- **누적 동작**: 같은 화면 내에서 [묻기] 누적 → 시스템 프롬프트에 이전 Q&A 포함하여 follow-up 자연스럽게 처리. 화면 전환 시 누적 초기화.
- **시각**: 동적 풀이 응답에 좌측 가는 잉크선 (해설자 발화임을 표시)

### 4.4 동적 풀이 RAG 연결
**니체 지식 RAG 연결됨**. HyDE 기반 검색 → 점수 임계값 통과 청크 → 시스템 프롬프트 주입.

응답 *맨 아래*에 옅은 출처 표기:
```
참고: 『즐거운 학문』 §125, 『차라투스트라』 서문 §3
```
- 11px, 색 #6b5f4f
- RAG 점수 임계값 통과한 청크 출처만

### 4.5 패널 닫기 / 화면 전환
- [해설 닫기] 버튼 또는 ESC → 슬라이드 아웃 (300ms)
- 패널 열려있는 동안 학습자가 *클릭으로 다음 단락* 막음
- 화면 전환(#2 → #3) 시 동적 풀이 누적 초기화

### 4.6 활성 화면
- Ep 1: #2, #3, #4 만 (정적 나레이션 화면)
- Ep 2: #1, #2, #3 만
- 인터랙션 화면(#5~#7, Ep 2 #4)에서는 [해설] 버튼 *없음*. 학습자가 사상 풀이를 원하면 차라투스트라에게 직접 물음 → 페르소나 sLLM이 풀이 RAG로 응답.

## 5. Ⅴ. RAG UI 표시 정책

### 5.1 인터랙션 모드
- **RAG 결과 완전 숨김**. 차라투스트라 발화에 청크가 *내용으로 녹아*들어가지만, 학습자 화면에는 *발화*만 보임.
- 출처 표시 X. 인용 따옴표 X. 작품 몰입 우선.
- 발표 시연용 디버그 모드(개발자 토글)에서만 RAG 청크 표시 — 학습자 UI에는 없음.

### 5.2 해설 모드
- **응답 맨 아래 옅은 출처**. §4.4 참조.
- 본문 풀이는 sLLM이 자연스럽게 녹임. 출처는 *어디서 더 읽을 수 있는지* 단서.

### 5.3 호출 중 신호
- **별도 표시 없음**. 학습자에게 *"검색 중..."* 같은 신호 X. 작품 몰입 깸.
- §3.5의 *3초 이상 지연 시 "…" 인디케이터*가 사실상 흡수.

### 5.4 개인화 RAG 신호
- **숨김**. 차라투스트라가 *"그대가 아까 X라 했지"* 식으로 자연스럽게 호명하면 그것이 신호.
- 별도 *"기억 호출 중"* 표시 X.

### 5.5 첫 발화 (개인화 RAG 빈 결과)
- 첫 발화 시점엔 학습자 발화 누적 0개라 RAG 빈 결과.
- 시스템 프롬프트가 처리 — 차라투스트라가 *첫 만남 결*로 응답.
- UI 차원 별도 처리 X.

### 5.6 RAG 실패
- **투명 처리**. 차라투스트라는 RAG 없이 응답.
- 학습자 화면에 차이 없음. 에러 토스트 X.

### 5.7 sLLM 호출 자체 실패
- 별도 — 토스트 *"길이 잠시 끊겼습니다"* 또는 비슷한 결.

### 5.8 RAG 풀 범위 (백엔드 영역, UI 무관)
**기존 합의 일부 수정 — 단일 인덱스 + 시간 메타 필터링**:
- 인덱스에 차라 1부 + JW + BGE + GM + TI + EH 통합
- 메타 태그(`year`, `book`)로 검색 시 필터:
  - 인터랙션 모드: `year ≤ 1881` 필터 (시간 일관성)
  - 해설 모드: 필터 없음 (전체 — 학습 풍부함)

기존 `HANDOFF_CONTEXT.md` §4의 *"BGE/GM/TI/EH 제외"* 명시는 **인터랙션 모드 한정**으로 재해석. 해설 모드는 후기 저작 포함.

구현 디테일(임계값/청크 수/HyDE)은 다른 세션에서.

## 6. Ⅵ. 세이브 시스템

### 6.1 세이브 버튼 활성 화면
- **Ep 1**: #5, #6, #7 (인터랙션 화면)
- **Ep 2**: #4 (인터랙션 화면)
- 그 외 화면: 우상단 [세이브] 버튼 **표시하지 않음** (mockup §Ⅰ에서 muted 표시했으나 정책 확정 — 안 보임)

### 6.2 [세이브] 동작
- **첫 세이브** (기존 슬롯 비어있음): 모달 없이 즉시 저장. 토스트 *"저장되었습니다"* (1.8s)
- **덮어쓰기** (기존 슬롯 있음): 확인 모달
  ```
  저장
  ─────────────────────
  기존 저장 데이터를 덮어씁니다. 진행하시겠습니까?
  
  [취소]  [덮어쓰기]
  ```
  → [덮어쓰기] 클릭 시 토스트 *"저장되었습니다"*

### 6.3 [불러오기] 모달
타이틀 #1에서 [불러오기] 클릭 시:

**기억 없음**:
```
불러오기
─────────────────────
저장된 게임이 없습니다.

[돌아가기]
```

**기억 있음**:
```
불러오기
─────────────────────
Ep 1 — 화면 #7 시장 원경
2026-04-30  14:23

"그가 무엇을 짊어지고 왔는지 나는 알지 못한다.
다만 길을 함께 걸은 것뿐이다..."     ← summary 80자 미리보기, italic

[취소]  [불러오기]
```

### 6.4 모달 / 토스트 톤 정책
**평이한 시스템 톤**. 작품 결 시도 폐기.
- 제목: *"불러오기"*, *"저장"* (letter-spacing 강조 X, 일반 0.05em)
- 액션: *"불러오기"*, *"저장하기"*, *"덮어쓰기"*, *"취소"*, *"돌아가기"*
- 토스트: *"저장되었습니다"*

작품 결이 들어가는 곳은 별도(§6.5 transition, 차라투스트라 발화, 정적 나레이션, 해설). 시스템 UI는 평이.

### 6.5 Ep 1 → Ep 2 카운드오버 Transition
**3초 정적 + 백그라운드 요약 sLLM 호출.**

```
Ep 1 #8 [Ep 2로 계속] 클릭
↓
Frame 페이드 검은 배경 (600ms)
↓
잉크색 옅은 글씨 페이드인 (600ms):
   "밤이 깊었다.
    시간은 흘러, 시장의 새벽이 왔다."
↓
3초 정적 (이 동안 백그라운드에서 요약 sLLM 호출)
↓
페이드아웃 (600ms) → Ep 2 #1 시장 광장 페이드인 (600ms)
```

- transition 텍스트는 italic + 작품 결 유지 (시스템 메시지 아님 — 작품 내 시간 흐름 표현)
- 요약 sLLM이 3초 안에 안 끝나면: transition 화면 그대로 유지. 최대 30초 대기.
- **Failover**: 30초 초과 시 요약 *없이* Ep 2 진입. 학습자 화면 차이 없음.

### 6.6 불러오기 시 동작
1. 슬롯 모달 [불러오기] 클릭
2. 페이드아웃 → 저장된 `scene_index` 화면 페이드인
3. 자동 발화 sLLM 호출 (시스템 프롬프트에 *"# 이전 만남의 기억"* 주입 — `EP1_TEXT_AND_PROMPTS.md` §4-3)
4. 차라투스트라가 *재진입 결*로 발화 — sLLM이 톤 조절
5. 입력 활성

UI 차원 추가 알림 신호 X. 자동 발화가 시작되면 그것이 신호.

### 6.7 자동 세이브 X
[세이브] 버튼 클릭 시에만 요약 sLLM 호출. 학습자가 자기 기억을 *선택하는* 행위.

### 6.8 JSON schema / `recent_messages` 턴 수
백엔드 영역. 다른 세션에서 결정.
